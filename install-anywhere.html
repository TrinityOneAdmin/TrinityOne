Install anywhere

Install TrinityOne from any church.

Every TrinityOne church running its own relay is a complete copy of the app — the member app, the steward console, the Android APK, the Bibles. There is no single site this all depends on. If trinityone.church doesn't load where you are, install from a friend church's relay URL. Same app, same signatures, same network.

Every church is a mirror

A church's relay isn't just for chat — it also serves the website, the APK and the modules. Hundreds of churches, hundreds of install points.

By design, not a workaround

The app was built to travel with the people. Self-hosting churches don't just keep their own data — they keep TrinityOne reachable for everyone.

Same app, anywhere

An APK from a friend church is the same APK as on trinityone.church — same signing key, same code, byte-for-byte identical.

How to install from a church relay

1

Get a friend church's relay URL

Ask anyone already on TrinityOne — a friend, a partner church, a steward of your own church if it runs its own relay. The URL looks like one of these:

https://relay.somechurch.org  or  https://something.tailbeaac0.ts.net

It doesn't matter if the church is in your country, your network, your denomination — every TrinityOne relay serves the same files. You're not joining their church by installing from them; you're just downloading the app.

Don't know any TrinityOne church yet? Ask whoever shared this guide with you for the URL of any relay they can reach. That's the bootstrap.
2

Open or install from that URL

What you do next depends on the device. The relay URL you got in step 1 replaces trinityone.church in every link below.

Android phone

  • Open a browser (Chrome / Firefox / Brave).
  • Visit <relay-url>/trinityone.apk — the file downloads.
  • Open it; allow “install from unknown sources” if prompted.
  • Done — open TrinityOne and follow the wizard.

iPhone / iPad

  • Open Safari (not Chrome — only Safari can install PWAs on iOS).
  • Visit <relay-url>/
  • Tap the Share button → Add to Home Screen.
  • TrinityOne lives on your home screen like an app.

Any browser (web)

  • Visit <relay-url>/
  • That's it — the full member app runs in your browser. Nothing to install.
  • You can bookmark it; subsequent opens are offline-capable.

Steward console

  • Same URL with /steward.html on the end.
  • Or the steward APK: <relay-url>/trinityone-steward.apk.
  • The setup wizard runs first time. Identical to the main site.
Once installed, the app keeps working through TrinityOne's other relays automatically — you only need that one URL for the install itself.
3

(Optional) verify what you got

If you'd like to be sure the APK is the genuine one, you can compare its signing fingerprint with the one published by the project. The signing key is the same across every relay and every release — so a fingerprint match proves the APK hasn't been altered.

On Android, the package installer shows the signing-key fingerprint before install (under “app details” → “signing certificate”). Trustworthy churches sharing a TrinityOne APK already know it's the right one — but it's there to check if you'd like to.

A formal “verify” one-click flow is coming. For now: if the church sharing the APK is one you trust, you're fine. If you don't know them, ask in your usual circles whether anyone has cross-checked the fingerprint.

What this works around — and what it doesn't

Some honest limits, so nobody is surprised.

  • You still need internet to reach some relay URL. If every TrinityOne relay you know is blocked, you'll need a VPN, or someone outside the firewall sharing the APK as a file (Bluetooth, USB, Signal/Telegram attachment).
  • iPhone is the hardest case. Apple's design means a PWA must be installed via Safari at the relay's URL — there's no “sideload” equivalent. If every URL is blocked, a VPN is the practical answer for the first install only; the app runs normally after.
  • This is install-time. Once installed, the app is much harder to disrupt — chat, Scripture and modules ride over the relay network with multiple fallbacks. See the resilience design for how that works.

The point

TrinityOne is meant to be carried — not warehoused. Every church running a relay is keeping the app alive for everyone, not just for itself. If you're reading this because you couldn't reach the main site, that's a sign the design is doing its job: the app travels through its people, not through a server.

If you'd like to be a mirror for others

If you run a church on TrinityOne, you're already a mirror — your relay serves the app, the APK and the modules to anyone who points a browser at it. You don't have to do anything. Sharing your relay URL with another church (or an organisation that helps people in restricted regions) makes you part of the install mesh. More on running a relay →
